# 应用icon创作大师 PRD

## 1. 产品定位

应用icon创作大师是一款面向广告投放与买量团队的 H5 工具，用于把待投产品、竞品、参考图和投放约束，转化为可直接用于投放测试的移动应用 Icon 素材。

当前版本聚焦游戏类 App Icon，不处理 banner、信息流大图、视频素材等其他广告形态。

## 2. 目标用户

- 买量投放人员：需要快速生成多个可测试 Icon 方案。
- 广告创意策划：需要把产品卖点、竞品风格、平台限制转成生成提示词。
- 中小团队开发者：没有完整美术团队，但需要快速产出投放素材。
- 代理商/素材团队：需要沉淀可复用的 Prompt 模板与交付流程。

## 3. 核心价值

- 减少从产品理解到 Icon 生成的人工沟通成本。
- 将审美争议转化为投放可用性检查：小尺寸识别、主体清晰、平台安全。
- 支持参考图优先：用户上传图、主产品 Icon、竞品 Icon 作为生成依据。
- 支持结构化 Prompt 管控，便于后续迁移到 n8n、数据库和多模型系统。
- 支持多尺寸 PNG 导出，满足投放和商店素材交付。

## 4. 当前功能范围

### 4.1 S1 产品检索与需求输入

用户输入：

- 待投产品名称
- 竞品名称，最多 3 个
- 投放平台：Google Ads / Meta / TikTok
- 情绪目标，最多 3 个
- 是否添加文字角标
- 角标文字
- 用户参考图，最多 2 张

系统能力：

- 通过 Google Play 检索主产品和竞品。
- 解析主产品资料、Icon、Feature Graphic、截图。
- 解析竞品资料和竞品 Icon。
- 上传用户参考图并生成可访问地址。

### 4.2 S2 产品 / 竞品 / Icon 分析

系统调用文本/视觉理解模型，分析：

- 核心玩法/题材
- 美术画风/视觉风格
- 主要受众和核心卖点
- 主产品 Icon 与竞品 Icon 的画面内容、色彩、构图、文案特征
- 共性市场范式和差异化机会
- 可用于后续 Prompt 组装的字段

平台约束不依赖模型实时判断，采用可编辑规则导入。

### 4.2.1 竞品评分规则与看板

竞品评分用于判断用户输入或系统检索到的竞品是否适合进入后续 Icon 风格分析。

当前建议作为下一阶段功能预留，不影响现有 S1-S8 主流程。

评分维度包括：

- 分类匹配分值
- 商业模式与其他信号
- 本品/竞品名称关键词近似度

本品/竞品名称关键词近似度只用于加分或不加分，不设置跨品类负分项。

建议评分项：

- 核心关键词高度重合
- 关键词语义近似
- 关键词弱相关
- 名称关键词无关

所有单项分值应支持人工配置。自动入库、自动排除、人工审核的阈值也应支持人工配置。

配置调整后，已存竞品不应直接覆盖历史结果。系统应标记相关竞品为待更新，并支持：

- 用户手动更新评分
- 定时自动更新评分
- 每次更新使用当前最新配置
- 每次评分结果记录使用的配置版本与上次生成时间，精确到分钟

竞品看板至少展示：

- 竞品名称
- Icon
- 总分
- 分层结果：自动入库 / 人工审核 / 自动排除
- 使用配置版本
- 上次生成时间
- 是否待更新

### 4.3 S3 Prompt 组装与用户确认

系统基于 S1/S2 结果调用模型生成 Prompt 计划。

用户可操作：

- 选择生成数量，当前最多 2 个方案
- 选择创意方向：保守 / 点击强化 / 极致夸张
- 编辑 Prompt 模板
- 编辑最终生成 Prompt
- 确认 Prompt 后进入生成

生成策略：

- 有参考图：以参考图 + Prompt 的方式生成。
- 无参考图：以纯文本 Prompt 生成。

### 4.4 S4 Prompt 自检

系统调用模型检查 Prompt 中可能触发拒绝或平台风险的内容。

检查范围：

- 色情裸露、未成年人风险
- 血腥暴力、自残
- 仇恨、极端主义
- 违法行为、毒品、武器、诈骗
- 版权、商标、竞品 Logo 复制
- 误导性承诺、夸张效果承诺
- 规避平台审核或安全规则的措辞

### 4.5 S5 Icon 生成

系统使用 S4 生成并经用户确认的 Prompt 调用图像模型。

每个方案输出：

- 1 张透明背景 PNG Icon 原图
- 1 张场景展示图，用于预览手机桌面、Google Play、广告位效果

当前版本限制：

- 最多一次生成 2 个方案。
- 场景展示图仅用于前端预览，不进入最终 ZIP 交付包。

### 4.6 S6 自动质检与用户决策

用户可选择：

- SELECT：选择方案
- REGENERATE：进入重生
- EXPORT：导出已选方案

当前版本的质检能力偏轻量，后续需要增强真正的视觉反解和 64px 可读性评分。

### 4.7 S7 重生执行

用户可以基于原 Prompt 编辑后重新生成。

限制：

- 每个 Icon 最多重生 2 次。
- 超出后提示用户返回修改 Prompt 模板或前序分析结果。

### 4.8 S8 多尺寸导出

最终导出：

- PNG：1024 / 512 / 256 / 128 / 64
- ZIP 包：仅包含多尺寸 PNG

不导出：

- 场景展示 SVG
- manifest JSON

## 5. 非功能需求

- 可部署：支持 Railway 部署，未来支持对象存储和数据库。
- 可配置：模型、Prompt 模板、输出 Schema 放在 `icon-agent.config.json`。
- 可测试：提供 quick / ai / full smoke test。
- 可回滚：通过 Git commit 和 tag 管理版本。
- 安全：API Key 只放环境变量，不进入 GitHub。

## 6. 当前已知限制

- 本地文件目录 `uploads/`、`generated/`、`exports/` 不适合作为长期线上存储。
- Railway 重启或重新部署后，本地生成文件可能丢失。
- S2 视觉分析依赖模型对公网图片 URL 的支持。
- Google Play 抓取存在网络波动和页面结构变化风险。
- 场景图目前为模板合成，不是完整真实平台截图。

## 7. 成功指标

- 用户从输入产品到生成首批 Icon 的流程可完整跑通。
- 一次任务能稳定生成 1-2 个 Icon 方案。
- 导出 ZIP 中只包含标准 PNG 尺寸文件。
- Prompt 模板和模型配置可在不改核心代码的情况下调整。
- 线上部署版本可以通过 smoke quick 测试。
